# Instructions projet Kizz (à respecter)

## Contexte
- Projet existant : **ne jamais recréer le projet ni re-scaffolder Expo**.
- Travailler uniquement sur la base déjà en place (structure, dépendances, conventions, nomenclature).
- Avant tout changement Expo/React Native, vérifier la doc versionnée SDK 56 :  
  https://docs.expo.dev/versions/v56.0.0/

## Banque de questions
- La banque doit couvrir au minimum ces styles de jeu :
  - TTMC
  - Trivial Poursuite
  - Jeu TV
  - Question pour un champion
  - Pays & capitales (inclure placement de pays)
  - Carte de France dédiée (géographie France)
- Conserver des questions **courtes, claires, vérifiables**, en français.
- Éviter le contenu sous copyright reproduit mot pour mot ; produire des formulations originales.

## Règles de modification
- Faire des changements **chirurgicaux** dans les fichiers existants.
- Ne pas ajouter de dossiers/outils non demandés.
- Ne pas toucher aux fichiers non liés à la demande.
- Garder une UX simple : navigation entre questions, affichage/masquage de réponse, filtres de catégories.

## Git / branches
- Vérifier la branche courante avant de commit.
- Si le dépôt utilise plusieurs worktrees, éviter les confusions de branche.
- Pousser explicitement vers la branche demandée par l’utilisateur (ou la branche principale du dépôt).
